Operation Failures

7 sub-issues ▸

Operation triggered but stopped before producing output, or ran below expected performance.

AI Provider Unavailability & Rate Limits Very High

An AI provider (Anthropic/Claude, Google/Gemini, or OpenAI/GPT) was unreachable or rejecting requests due to concurrent volume. Provider downtime affects all operation types.

All models exhausted after N attempts · 429 Resource Exhausted — Vertex AI / Anthropic

Sub-variants: Claude down (very high), Gemini down (medium — Export Process only), AI Interviewer unavailable (low-medium — requires both Claude + GPT down), AI request timed out (low-medium).

Confirmed: Multiple customers Jan 27–28 (rate limit wave), KLA Export Jan 21, Google Finance R2R, EY training video Mar 3. Abhinav: "Gemini model is down." Saideep: "Claude had a downtime."
All Operations Export Process → Gemini AI Interviewer → Claude+GPT

Fix: Wait for provider to recover, re-run. Check status.anthropic.com / Google Cloud status. Spread operations out to avoid rate limits.

Execution Timeout Medium

Operation exceeded the ~90-minute run limit. Root cause is typically a misconfigured output token cap (e.g. set to 8,192) causing the model to retry indefinitely.

Confirmed: Google Finance R2R (Feb 26), EY training video (Mar 3).
Process CaptureGenerate Document

Fix: Engineering to check token cap config. Re-run after fix.

Token / Context Limit Exceeded Medium

Uploaded file converts to more text than the model's context window allows (over 950,000 tokens). Files can appear small but still exceed limits — a dense Excel with many rows or a bundled PDF are common triggers.

TokenLimitException: N tokens in input, but max input tokens is 950000 · Max tokens exceeded during tool use, arguments were truncated
Confirmed: KLA UDC Invoice Mar 17 (Kimball: "context limit issue, a fix has been pushed"), Google Finance R2R Feb 26–27.
Update DocumentGenerate DocumentProcess Capture

Fix: Customer to split document into smaller sections. Engineering to review chunking options.

Infrastructure Concurrency Failure Low

Backend failure from too many operations running simultaneously — either a database key conflict (race condition) or the DB running out of connection slots.

UniqueViolation: duplicate key value violates unique constraint artifact_input_artifacts_pkey · QueuePool limit of size 10 overflow 25 reached
Confirmed: Tyler Tech POV Feb 19 (Kimball: "race condition"). Write conflict fix deployed Feb 19. DB pool: Linear CUS-195.
All Operations (system-wide)

Fix: Race condition fix deployed. DB pool — engineering to increase pool size. Advise customers not to upload multiple docs simultaneously.

Operation Initiation Failure Low

The run never started — no output, no error shown to the customer. Applies to any operation type. Typically a workspace configuration or provisioning issue on the backend.

All Operations

Fix: Escalate to Engineering. Check workspace provisioning status.

Platform Performance Degradation Medium

Platform-wide slowness — operations take longer than expected, session success rates drop, or Companion/Interviewer response times are noticeably degraded. Distinct from a full outage: the platform is running but underperforming.

Confirmed: Astera Labs lag, KKR POV (1 in 12 sessions succeeding), AI Interview lag across multiple workspaces.
All Operations

Fix: Escalate to Engineering with timestamps. Check if coincides with AI provider degradation.

Interviewer Audio Input Failure Medium

Microphone or audio is not functioning during an AI Interviewer or Companion session — the system cannot hear the user, mic permissions were reset mid-session, audio is not transmitted over Teams/Zoom screen share.

Confirmed: AAK (ZD Mar 2), multiple Teams screen-share users, UMB and EDF demo sessions.
AI InterviewerCompanion

Fix: Check browser mic permissions, ensure not using Teams/Zoom audio passthrough. Restart session.

AI Output Quality

8 sub-issues ▸

The operation completed, but what the AI generated is wrong, inconsistent, or incomplete.

Content Hallucination High

AI produced factually incorrect, fabricated, or contextually irrelevant content in a process capture, SOP narrative, or Advisor recommendation. Common when source material is ambiguous, too long, or of poor quality.

Confirmed: EDF (random info pulled into output), PubMatic (inconsistent employee designations), Aven Hospitality AI Interviewer Q&A.
Process CaptureGenerate DocumentAdvisorCoach

Fix: Customer to review output and re-run with clearer source material. Escalate persistent cases to Engineering.

AI Reference Hallucination (Section ID / Resource Key) Medium

AI invented an identifier that does not exist in the workspace — a section ID during the critique step (e.g. "section_one" instead of "1") or a diagram node resource key.

No section found with name section_one · NotFoundException: Process with resource key XXXX not found in workspace YYY
Confirmed: Mar 22 (section ID — Satyendra: "this is LLM hallucination"), Mar 2 (resource key).

Fix: Re-run operation. Engineering investigating prompt tuning for section ID format.

Process Flow Diagram Failure Low

Generated process diagram has incorrect layout, overlapping swim lanes, missing or extra nodes, or a Mermaid syntax error preventing rendering. Operation may show as successful while the diagram is absent or malformed.

Confirmed: Larvin V ticket (Mar 20).

Fix: Re-run diagram generation. Report swim lane configuration issues to Engineering.

Document Export Discrepancy Medium

Content visible in the Architect UI differs from what is produced in the exported document — data missing from PDF tables, blue formatting lines in Word exports, incorrect content rendering.

Confirmed: McKesson Advisor Export (ARC-4920, Mar 11), PubMatic blue lines in narrative (ZD #11403, Mar 24).

Fix: Escalate to Engineering with exported file and screenshot of UI content.

Incomplete Operation Output Medium

Output was produced but is incomplete — sections missing, content cut off, or document ends abruptly. Also covers operations that show as completed in the UI but produce no artifact.

Confirmed: Multiple "Operations showing complete but not producing anything" tickets (Mar 17).

Fix: Re-run. If persistent, escalate to Engineering with operation ID.

Interviewer Response Not Captured Medium

AI Interviewer session ran but responses were not correctly recorded — responses appear empty, are mis-attributed, or the AI made transcription errors (e.g. accent misrecognition).

Confirmed: South Asian accent errors (ZD #11 Mar 18), AI Interview getting words wrong (ZERO vs ZFRO).

Fix: Re-run interview. For accent issues, recommend clear audio input.

Process Step Screenshot Failure High

Companion or Interviewer session completed but process steps are missing screenshots — none were extracted, or incorrect screenshots captured (e.g. Klarity's own UI instead of customer's workflow).

Confirmed: AAK (multiple tickets), Forvis Mazars, ServiceNow, Presidio, Keysight — 8+ tickets across 6+ customers.

Fix: Escalate to Engineering. Known issue across multiple customers.

Non-Deterministic Output Low

Re-running the same operation produces noticeably different results each time. This is expected LLM behavior — the model introduces natural variation across runs.

Talk Track: "This is how AI models work — they generate text probabilistically. Each run is unique. The best approach is to run the operation, review the output, and use the editing tools to refine."

File & Template Errors

8 sub-issues ▸

The source file, upload, or template is the problem — not what the AI produced, but what it was given to work with.

File Size Limit Exceeded Medium

File is too large to process — at upload (input) or when saving generated output. Input: large video recordings (2.5 GB+) and dense Excel files. Output: AWS S3 max size.

Chat failed: Your files are too large for us to process · EntityTooLarge: Your proposed upload exceeds the maximum allowed size
Confirmed: JLL India Commission (input), Ripple Pro Service Accruals (output).

Fix: Customer to compress or split file. For output errors, escalate to Engineering.

Unsupported File Format Low

Architect could not open or parse the uploaded file — unsupported format, password-protected, or corrupted. Supported: .docx, .xlsx, .pdf (unencrypted).

KlarityException: Cannot parse artifact XXXXX
Confirmed: Ripple OTC Revenue, Presidio workspace (Feb 23).

Fix: Customer to re-export as PDF or supported format. Remove password protection.

File Preprocessing Timeout / OCR Corruption Low

File entered the pipeline but failed before being ready — either preprocessing timeout (40+ min) or OCR corruption (null characters in scanned PDFs).

TimeoutError: Timeout waiting for artifact XXXXX · Null characters from OCR causing DB insert failure
Confirmed: UMB Mapping (40 min), Google PDF (Feb 13). OCR fix deployed Feb 13.

Fix: Check file integrity, re-upload. OCR fix deployed Feb 13.

Spreadsheet Layout Not Recognized Low

Excel file layout didn't match expectations — wrong column positions, missing headers, or extra blank rows.

IndexError: single positional indexer is out-of-bounds (excel_service.py)
Confirmed: Genpact Spreadsheet.

Fix: Customer to clean up spreadsheet format. Escalate to Engineering.

Output File Too Large to Save Low

Operation completed successfully but the generated output was too large to upload to AWS S3.

EntityTooLarge: PutObject exceeds maximum allowed size
Confirmed: Ripple Pro Service Accruals (30 min).

Fix: Escalate to Engineering. Output size limits may need adjustment.

Export Template Misconfigured Low

Export Process failed because the template references a section that doesn't exist in the customer's workspace.

No section in template with ID section_one
Confirmed: AAK Shipment Cost (Mar 19), Uber-Logistics template inconsistency (ZD #11306, Mar 23).

Fix: Update export template to match workspace process structure.

Missing Process Node Low

Operation tried to access a process step or hierarchy node that no longer exists — deleted, renamed, or never configured.

NotFoundException: Process not found in workspace · Hierarchy node not found in workspace
Confirmed: Aven Hospitality (Feb 26), ServiceNow Deloitte forecast.

Fix: Restore or recreate the missing node. Re-run.

FFmpeg / Video Conversion Failure Low

Server couldn't convert session recording from WebM to MP4. Usually a corrupted recording or codec issue.

RuntimeError: FFmpeg conversion failed! See logs for details.

Fix: Customer to restart session and try again. Escalate to Engineering with session timestamp.

Access & Authentication

7 sub-issues ▸

Customer cannot log in, cannot provision team members, or workspace access is not working as expected. See the Access & Auth Playbook tab for detailed troubleshooting flows.

Workspace Not Provisioned High (Blocker)

A new customer or user cannot access their Architect workspace — no workspace exists for them yet. Blocks all other work until provisioning is complete.

Fix: Submit internal work request for provisioning. Typical turnaround: same day.

SSO / OIDC Integration Failure High

Domain-wide authentication failure — all users in the customer domain unable to log in. OIDC integration errors, expired client secrets, misconfigured redirect URIs.

Confirmed: McKesson Okta migration (Mar 12), ZD #10731.

Key Gotchas: Okta client secrets expire after set period. Redirect URIs must exactly match. Token endpoint auth method must be "client_secret_post". See Playbook tab for full Okta/Azure AD troubleshooting.

Authentication Loop Medium

User logs in successfully but is immediately signed out and redirected to the login screen in a loop. Typically caused by session token expiry or SSO configuration mismatch.

Fix: Clear browser cookies for app.klarity.com. Check Chrome → Settings → Privacy → "Allow all cookies" (or whitelist Klarity). If SSO, verify the OIDC config with the customer's IT team.

SSO & Magic Auth Issues (Individual User) Medium

Individual user login failures: SSO access denied, Magic Auth link not arriving or expired, email case sensitivity mismatch.

Confirmed: Magic Link case sensitivity (Mar 20), ZD #10731.

Magic Auth common fixes: Check spam/junk, verify email matches exactly (case-sensitive), link expires in 10 min, must use same browser where link was requested. See Playbook tab.

User Provisioning & RBAC Low

Workspace admin cannot invite new users, or role/permission changes not taking effect after being saved.

Confirmed: UMB users seeing both Architect and Automate on Okta (Feb 26), multiple account activation requests.

Fix: Check Okta app assignments. For RBAC, clear cache and re-save permissions. Escalate to Engineering if persists.

Account Locked / Deactivated Low

User account is locked out, deactivated, or showing as inactive, preventing access. May be due to IdP JIT provisioning failure.

Fix: Check user status in Architect MiniApp. Reactivate if needed. Verify IdP is sending correct user attributes.

Interview Invite Delivery Failure High

Errors when dispatching bulk AI Interviewer invitations — invite emails fail to send, recipients don't receive them, or sender has no visibility into delivery status.

Confirmed: UMB (URGENT — ZD Feb 28), KLA pilot (Feb 24).

Fix: Escalate to Engineering immediately. Check email delivery logs. Verify recipient email addresses.

Product & Environment Failures

11 sub-issues ▸

The Architect workspace or a product feature is not behaving correctly — data not surfacing, features unresponsive, or the environment in an inconsistent state.

Process Index Visibility Failure Medium

Content that exists on the backend is not surfacing in the Process Index — completed sessions don't appear in search, uploaded files have disappeared. Re-running typically triggers re-indexing.

Confirmed: Freshworks (ZD #11004), Ardent Mills (ZD #11127), Sessions Contributed tab issues (Mar 14–16).

Fix: Re-run the operation to trigger re-indexing. If persistent, escalate to Engineering.

Process Hierarchy Mismatch Medium

Processes appear in the wrong place in the hierarchy, nodes ordered incorrectly, or Companion session updated the wrong parent node.

Confirmed: ZD #10663, Weird Ordering of Index Nodes (Mar 19), KLA Coach contribution not categorized (Mar 17).
Signals / Insights Dashboard Not Loading Medium

Insights dashboard is empty or showing errors despite data being present.

Confirmed: ZD #10714, #10718, Insights Table Formatting Issue (Feb 20).

Companion chat not loading, not responding, or producing no process output after a full session. May be caused by stale browser session or backend restart.

Confirmed: Linear ARC-5111 (Mar 24), Coach not working (Feb 25), multiple unresponsive session reports.

Fix: Hard refresh browser. Clear cache. If unresolved, escalate with session ID.

Process Step Save Failure (Network Error) Medium

Edits to process steps not being saved. Save appears to complete in UI but backend write fails silently; changes lost on refresh.

Confirmed: KLA-SP11 (Mar 10), Aven Hospitality (ZD #10615, Mar 3), AAK Network Error (Feb 26).
Post-Migration Workspace Failure Medium

After workspace/product migration, operations fail, files inaccessible, or configs don't apply correctly. Workspace may be on older config post-migration.

Confirmed: JLL post-migration (ARC-4965, ZD — Mar 17), V2 hierarchy export failure (Mar 13).
UI Status / Progress Inaccuracy Low

Interface communicates incorrect state — no status indicator while operation runs, incorrect error messages, or inaccurate progress percentage.

Confirmed: "Files too large" error when real cause was 429 rate limit (Mar 12), Inaccurate Completion Progress % (ZD Mar 19).
Phantom Companion Session Medium

A Companion/Coach session appears as "active" even after user ended it. Blocks new sessions. Requires backend session reset.

Confirmed: Ardent Mills (multiple users, ZD #11007), Calvetti Ferguson, KKR POV.

Fix: Escalate to Engineering for backend session reset.

Session Not Syncing to Process Index Medium

Completed session content doesn't appear in the Process Index — the relevant process node is not updated. Distinct from visibility issues; the data was not written at all.

Confirmed: Freshworks (ZD #11004), Ardent Mills worker queue backlog (Mar 13).
Advisor Insights Failure Medium

Advisor feature non-functional or producing incorrect output — can't be accessed, fails to generate, returns chat error, or uses outdated data snapshot.

Confirmed: McKesson (ARC-4670), Athene, Aven Hospitality (ZD Mar 2), Advisor Version Mismatch (ARC-4553, Feb 18).
Workspace Data Loss Critical

Workspace content unexpectedly deleted or overwritten. More severe and potentially irreversible than a save failure. Requires backend restore from S3 or database backup.

Confirmed: Stripe workspace wiped clean (ZD Mar 17), Athene existing process data overwritten, Keysight deal packet S3 deletion request (Mar 19).

Fix: ESCALATE IMMEDIATELY to Engineering. Request S3/DB restore. Document exact scope of loss.

Issue Triage Decision Tree
Step-by-step triage flow for incoming tickets and Slack reports. Follow from top to classify and resolve.

Step 1 — Where did the report come from?

Slack #ka-failed-operations → Go to Step 2 (Operation Failure)
Slack #architect-bugs-and-issues → Go to Step 3 (Bug Report)
Zendesk ticket → Go to Step 4 (Customer Report)
Customer Slack channel → Go to Step 4
↓

Step 2 — Operation Failure (#ka-failed-operations)

Click the "View flow run details" link → Find the error message in the thread (red ❌ from klarity-bot) → Match to Error Codes tab.

Error contains "models exhausted" / "429" / "timed out" → Operation Failures: AI Provider
Error contains "TokenLimit" / "Max tokens" → Operation Failures: Token Limit
Error contains "Cannot parse" / "EntityTooLarge" / "Timeout waiting" → File & Template Errors
Error contains "UniqueViolation" / "QueuePool" → Operation Failures: Infrastructure
Error not recognized → Escalate to Engineering with full error text
↓

Step 3 — Bug Report (#architect-bugs-and-issues)

Determine the symptom category:

"Can't log in" / "SSO error" / "Magic link" → Access & Auth (see Playbook tab)
"Output is wrong" / "Hallucination" / "Missing content" → AI Output Quality
"Can't see my process" / "Dashboard empty" / "Session stuck" → Product & Environment
"Network error" / "Save failed" / "Data gone" → Product & Environment: Save/Data
↓

Step 4 — Customer Report (Zendesk / Slack)

Gather information using the Ticket Intake checklist, then classify:

Customer can't access the platform at all → Access & Auth
Customer ran an operation and it failed → Operation Failure or File Error
Customer ran an operation and output is wrong → AI Output Quality
Platform feature is broken or not loading → Product & Environment
↓

Step 5 — Escalation Decision

Can CX resolve? Re-run, advise customer (file format, split docs, clear cache) → Resolve and close
Needs Engineering? Check Linear for existing ticket → Attach to existing or create new → Tag POC owner
Critical / Data Loss? Escalate IMMEDIATELY → Ping #architect-bugs-and-issues + Engineering lead
↓

Step 6 — Engineering Escalation Routing

Access & Authentication Playbook
Detailed troubleshooting for login, SSO, Magic Auth, and user provisioning issues. Sourced from the User Login & Onboarding Troubleshooting Guide, Entra ID SSO Config, and Magic Auth Setup Guide.

🔐 Authentication Methods Overview

Klarity Architect supports three login methods, now powered by WorkOS as the underlying auth platform:

  • SSO (Okta / Azure AD Entra): Enterprise standard. Uses OIDC with PKCE via WorkOS. Requires customer IT setup. Recommended for all enterprise accounts.
  • Magic Auth (Passwordless): One-click magic link sent to email via WorkOS MagicLink (migrated from Okta MagicLink). No passwords needed. Good for POVs, external interviewees, quick onboarding. Link expires in 10 minutes. OTP (one-time passcode) also supported.
  • Email / Password: Traditional login. Used as fallback. Requires manual account creation in Architect MiniApp.

🔄 WorkOS — Auth Platform Migration

Klarity is migrating its authentication layer from direct Okta OIDC integration to WorkOS — a developer-focused identity platform that provides a unified API for SSO, Magic Links, OTP, and organization management.

  • What changed: Auth flows now route through WorkOS instead of directly to Okta/Azure AD. This adds a consistent auth layer regardless of the customer's IdP.
  • Organization onboarding: New orgs are provisioned through WorkOS with E2E onboarding flow (see Loom: "WorkOS End to End Organization Onboarding").
  • MagicLink migration: Okta MagicLink → WorkOS MagicLink, with OTP support added (see Loom: "Okta/MagicLink to WorkOS MagicLink Migration, OTP Setup").
  • MiniApp integration: Creating and updating organizations now uses the MiniApp → WorkOS workflow (see Loom: "Creating & Updating Organizations in MiniApp Workflow").
  • Impact on troubleshooting: Auth errors may now surface WorkOS-specific messages. The underlying SSO config (Okta/Azure AD) remains customer-side, but the middleware layer is WorkOS.
Note: During the migration period, some customers may still be on the old Okta-direct flow while others are on WorkOS. Check which auth path the customer is on before troubleshooting.

Uncommon Client Secret Expired All users at org suddenly can't log in Okta/Azure client secret reached its expiration date. Customer IT to rotate secret in IdP. Klarity backend to update stored secret.
Uncommon Email Domain Mismatch SSO works for some users but not others User's email domain doesn't match the configured SSO domain. E.g. tylerfederal.com vs tylertech.com token mismatch. Add all relevant email domains to the SSO configuration. Tyler Tech case study: had to add both domains.
Uncommon Email Case Sensitivity User can't log in despite correct email Klarity treats emails as case-sensitive in some auth flows. "John@acme.com" ≠ "john@acme.com". Ensure email in IdP matches exactly. If mismatch exists, update in Architect MiniApp.

🔧 Okta SSO Setup Checklist

  • App Type: OIDC - OpenID Connect → Web Application
  • Grant Type: Authorization Code with PKCE
  • Sign-in Redirect URI: Must exactly match Klarity's provided URI (no trailing slash differences)
  • Token Endpoint Auth: Must be set to "client_secret_post" (not "client_secret_basic")
  • Scopes: openid, profile, email
  • User Assignment: Users or groups must be assigned to the Klarity app in Okta
  • Client Secret: Note expiration date, set calendar reminder 30 days before

🔧 Azure AD (Entra ID) SSO Setup Checklist

  • App Registration: Azure Portal → App registrations → New registration
  • Redirect URI: Platform = Web, URI = exact Klarity redirect
  • Authentication: Enable ID tokens, configure front-channel logout URL
  • Certificates & Secrets: Create client secret, note expiry date
  • API Permissions: Microsoft Graph → User.Read (delegated)
  • Users & Groups: Assign users/groups to the enterprise application

✉️ Magic Auth — How It Works

  • User enters email → Klarity sends a one-time magic link to that email
  • User clicks the link → automatically logged in (no password)
  • Link expires in 10 minutes
  • Must be opened in the same browser where it was requested
  • Email matching is case-sensitive in some flows
  • Great for: POV participants, external interviewees, quick demos
GTM Talk Track: "Magic Auth removes the #1 barrier to POV adoption — IT involvement. Your team can start using Architect in minutes with just their email, no SSO setup required. When you're ready for full rollout, we'll set up SSO."

🛠️ Architect MiniApp — Admin Operations

The MiniApp is the internal admin tool for workspace and user management:

  • Create/manage workspaces and organizations
  • Add/deactivate users
  • Reset user sessions (fix phantom sessions)
  • Check user authentication status
  • Update OIDC configuration
  • View workspace provisioning status
Eng Wishlist: MiniApp currently lacks: bulk user management, audit log visibility, self-service secret rotation, automated provisioning triggers. These are planned improvements.

🏢 Workspace Creation, Addition & Modification

  • Creating a new workspace: Done via Architect MiniApp → Create Workspace. Requires: customer organization, workspace name, desired configuration. Provisioning is typically same-day.
  • Adding users to a workspace: Via MiniApp (manual) or JIT provisioning (automatic on first SSO login). For Okta: users must be assigned to the Klarity app in Okta Admin → Applications.
  • Modifying workspace settings: Run configs, process hierarchy structure, and export templates are modifiable by workspace admins. Known issue: some settings may not persist on save (config persistence failure).
  • Workspace migration (V1 → V2): Managed by Engineering. Post-migration checks required: verify operations work, files accessible, configs applied correctly. Common failures: old config persisting, export template breaks.
  • Multi-workspace customers: Some customers (e.g., UMB) have both Architect and Automate on Okta — users may see both apps. Ensure correct app assignment in the IdP.
  • Workspace data restore: If workspace data is lost (critical issue), Engineering can restore from S3 snapshots or database backup. Requires: exact scope of loss, workspace ID, timestamp of incident.

📧 Interview Invite Delivery Troubleshooting

  • Symptom: Bulk AI Interviewer invites sent via "Assign Interview Task" but recipients don't receive them.
  • Common causes: Email delivery failures (org email filters, spam), recipient email mismatch, blast saturation (100+ invites overwhelming the system).
  • Troubleshooting steps: Check email delivery logs with Engineering. Verify all recipient emails. Send in smaller batches (recommended: 20-30 at a time). Check recipient spam/junk folders.
  • Known issue: Sender has no visibility into which invites were successfully delivered — this is a UX gap being tracked.
  • Confirmed cases: UMB (URGENT P0, ZD Feb 28), KLA pilot (Feb 24).

Feature Glossary
Key Klarity Architect product terms, features, and concepts for all internal teams.

AI Interviewer

AI-powered interview agent that conducts process capture conversations with users. Uses Claude + GPT models. Records sessions, extracts process steps, and captures screenshots. POC: Aditya.

Coach

AI coaching feature that guides users through process improvement. Provides recommendations, identifies gaps, and suggests optimizations based on captured process data. POC: Aditya.

Companion (DPI Companion)

In-app chat assistant for the Process Index. Always-on screen recording during sessions. Captures process steps as users work. Can create phantom sessions if not properly ended. POC: Abhinav.

Advisor

Analytics and recommendations engine. Generates insights from captured process data. Can fail if using outdated data snapshot. Related: ARC-4670, ARC-4553. POC: Abhinav.

Operations

Core processing engine — runs Process Capture, Generate Document, Update Document, Export Process, Enrich Spreadsheet. Failures tracked in #ka-failed-operations. POC: Abhinav.

Process Index

Hierarchical view of all captured processes in a workspace. Supports search. Background indexing job runs after operations complete. Visibility issues are a known category.

Process Hierarchy

The organizational structure of processes — parent nodes, child nodes, groups. Can become mismatched after migrations or concurrent edits. POC: Kimball.

DPI Agent

Document Processing Intelligence agent. Powers the AI pipeline for document analysis. Three-Factor Alignment Test validates output quality. POC: Kimball.

Context Store

Customer-configurable rules and business terminology that guide AI behavior. Misconfiguration can cause subtle output errors that look like hallucinations.

Magic Auth

Passwordless authentication via one-time magic link sent to email. Link expires in 10 min. Must use same browser. Case-sensitive email matching. Ideal for POVs and external users.

SSO / OIDC

Single Sign-On using OpenID Connect protocol with PKCE. Supported IdPs: Okta, Azure AD (Entra ID), Google Workspace. Requires customer IT setup. Enterprise standard.

Architect MiniApp

Internal admin tool for workspace and user management. Create workspaces, add/deactivate users, reset sessions, update OIDC config. Currently lacks bulk management and audit logs.

Signals / Insights Dashboard

Analytics dashboard showing process intelligence and operational metrics. Known issue: can show empty despite data being present (ZD #10714, #10718).

Export Process

Generates output documents from captured process data using templates. Uses Gemini AI (not Claude). Fails if template section IDs don't match workspace structure.

Generate Document

Creates new documents (SOPs, narratives) from process data. Uses Claude. Subject to token limits (950K max). Can produce incomplete output.

Enrich Spreadsheet

AI-powered spreadsheet analysis and enrichment. Sensitive to spreadsheet layout — unexpected column positions or missing headers cause crashes (excel_service.py).

#ka-failed-operations

Slack channel where all operation failures are posted automatically by klarity-bot. Primary source for triage. Each failure has a "View flow run details" link to #ka-operation-notifications.

#architect-bugs-and-issues

Slack channel for bug reports from internal teams. Covers all issue types including access, UI, and product behavior issues.

Linear

Engineering project management tool. Bug tickets are created as Linear issues with ARC- prefix (e.g., ARC-4920). CUS- prefix for customer-reported infrastructure issues.

Three-Factor Alignment Test

Quality validation framework for DPI Agent output. Tests alignment across three dimensions to ensure generated content matches source material.

Workspace

A customer's isolated environment in Architect containing their processes, documents, operations, and configurations. Must be provisioned by Klarity before use. Workspace data loss is a Critical-severity issue requiring S3/DB restore.

Workspace Provisioning

The process of creating a new workspace for a customer. Done via Architect MiniApp. Blocks all other work until complete. Common first step in onboarding. Provisioning failures categorized under Access & Authentication.

Organization

A top-level entity in Architect that can contain multiple workspaces. Organizations map to customer companies. Users are assigned to organizations, and their access is controlled at the org and workspace level.

User Provisioning (JIT / SCIM)

Just-In-Time provisioning creates user accounts automatically on first SSO login. SCIM (System for Cross-domain Identity Management) enables automated user sync from IdP to Architect. Both can fail if attribute mapping is misconfigured.

RBAC (Role-Based Access Control)

Permission system controlling what users can do within a workspace. Roles include admin, editor, viewer. Permission changes can fail to persist (known issue). Future: fine-grained access control predicted to cause misconfiguration tickets.

Workspace Migration

Moving a workspace between platform versions (e.g., V1 → V2). Post-migration failures are a known issue category — operations may fail, files become inaccessible, or old configs persist. Confirmed: JLL (ARC-4965).

Config Persistence

Workspace settings and run configurations that should persist across sessions. Known issue: settings sometimes revert to previous values on page refresh. Backend write appears to succeed but change is not actually stored.

Assign Interview Task

Operation to send bulk AI Interviewer invitations to users. Known delivery failures at scale — invites fail silently with no visibility into which succeeded. Confirmed: UMB (P0), KLA pilot.

OIDC / PKCE

OpenID Connect with Proof Key for Code Exchange — the auth protocol used by Klarity for SSO. Requires: redirect URI match, client_secret_post auth method, openid/profile/email scopes. Client secrets expire.

WorkOS

Developer-focused identity and auth platform that Klarity is migrating to. Provides unified API for SSO, Magic Links, OTP, and organization management. Replaces direct Okta OIDC integration as the auth middleware layer. See #architect-product-updates for migration details and Loom walkthroughs.

WorkOS MagicLink

Passwordless auth via WorkOS (replacing Okta MagicLink). Sends a one-time link or OTP code to the user's email. OTP support is a new addition post-migration. Link expires in 10 minutes, must use same browser.

WorkOS Organization

A WorkOS entity representing a customer organization. Created during onboarding via MiniApp → WorkOS flow. Maps to a Klarity organization. SSO connections and user directories are configured at the WorkOS org level.

ABAC (Attribute-Based Access Control)

Next-generation access control model in development for Architect. Goes beyond RBAC by evaluating user attributes, resource attributes, and environmental conditions for access decisions. JLL has specifically inquired about timeline. Expected to enable fine-grained permissions.

Dual-Screen Companion

Enhancement to Companion launched Apr 8, 2026. Allows Companion to observe both screens during coaching sessions — primary workflow plus supporting reference material. Critical for multi-monitor enterprise users who reference secondary systems while executing core processes.

V2 Migration

Platform upgrade from V1 to V2 architecture. Known issues: empty leaf nodes converting to folder nodes post-migration (DCG, BCG affected), operation failures on migrated workspaces, old configs persisting. Confirmed issue: JLL (ARC-4965). Post-migration validation required.

"What's New" Sidebar

In-app feature launched Mar 28, 2026 to surface product updates to users. Shows gradient highlight + badge for new features, plain display when caught up. Created because major features (Companion, Proactive Advisor, Teams, on-demand diagrams) were being discovered by accident months after launch.

UIP (Unstructured Input Processing)

Pipeline that processes unstructured inputs (documents, videos, audio) into structured process data. Failures tracked via automated Slack detection in #ka-failed-operations. High volume — 5+ failures per day during peak periods. Common cause of Zendesk ticket clusters.
